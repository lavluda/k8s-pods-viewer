# Contributing

Thanks for taking the time to contribute to `k8s-pods-viewer`.

## Before opening a pull request

1. Start from the latest `main` branch.
2. Keep changes focused on a single problem.
3. Run the local checks before sending the PR.

## Local checks

```bash
go test ./pkg/... ./cmd/...
make build
```

## Pull requests

- Include a short description of the problem and the change.
- Add screenshots or terminal output if the UI changed.
- Mention any follow-up work that is intentionally left out.

## Issues

Bug reports and feature requests are welcome through GitHub Issues.

## Security

If you find a security issue, please contact the repository owner privately
before opening a public issue.
