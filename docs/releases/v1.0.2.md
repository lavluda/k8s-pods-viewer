# k8s-pods-viewer v1.0.2

`k8s-pods-viewer` v1.0.2 improves resilience and makes live pod usage easier to read.

## Highlights

- watch disconnects now auto-heal cleanly without corrupting the TUI
- transient Kubernetes watch and metrics reconnect issues are surfaced as in-app status messages instead of raw terminal log spam
- pod rows now show actual `used` values alongside `req/lim`
- pod usage bars now fall back to requests when limits are not set, so request-only workloads no longer look completely idle
- binary memory values now promote units once the current unit would exceed `10000`

## Release validation

This release was validated with:

- automated Go tests via `go test ./pkg/... ./cmd/...`
- release smoke checks via `make test-release`
- local build verification via `go build ./cmd/k8s-pods-viewer`

## Notes

- best-effort pods without requests or limits still show actual `used` values, but their bars remain unscaled because there is no declared baseline
- live pod and node usage still depends on `metrics.k8s.io` being available in the cluster

## Install

```bash
go install github.com/lavluda/k8s-pods-viewer/cmd/k8s-pods-viewer@v1.0.2
```
