# k8s-pods-viewer v1.0.0

`k8s-pods-viewer` is now ready for general use as a terminal UI for live Kubernetes pod visibility.

## Highlights

- Live `cpu` and `memory` pod bars sourced from `metrics.k8s.io`
- Pod bars scale against each pod's configured resource limits
- Pod value labels show `request / limit`
- Right-side node panel shows:
  - live node CPU and memory usage
  - assigned pod requests
  - node allocatable capacity
- Namespace, pod-selector, and node-selector filtering
- Pod sorting by usage, creation time, namespace, name, node, and phase
- Default refresh interval of `1s`

## Release validation

This release was validated with:

- automated CLI flag and config precedence tests
- release smoke checks via `make test-release`
- a live k3s cluster smoke pass on a 3-node cluster with Metrics Server enabled

Live-cluster checks covered:

- default view rendering
- `--namespace`
- `--pod-selector`
- `--node-selector`
- `--pod-sort`
- `--resources`
- custom `--style`
- `--context`

## Notes

- `metrics.k8s.io` support is required for live usage bars
- pod graphs use live usage, while pod text values show requested resources against limits
- this project is Apache-2.0 licensed and includes code derived from `awslabs/eks-node-viewer`
