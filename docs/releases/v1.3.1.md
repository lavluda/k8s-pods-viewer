# k8s-pods-viewer v1.3.1

`k8s-pods-viewer` v1.3.1 is a small navigation stability release for busy clusters where live CPU or memory sorting can otherwise reshuffle rows while an operator is using the keyboard.

## Highlights

- keyboard input now pauses automatic pod reordering for 30 seconds, keeping the currently rendered order steady while moving the selection or opening actions
- newly visible pods are still appended in the active sort order during the pause, so the list remains current without disrupting the rows already on screen
- changing the configured pod sort clears the remembered order immediately, so explicit sort changes still take effect right away
- the pod list refresh cadence is now documented as `2s`, matching the current runtime behavior
- regression coverage now verifies that auto-sorted rows stay stable during keyboard navigation and resume sorting after the pause expires

## Release validation

This release was validated with:

- automated Go tests via `go test ./cmd/... ./pkg/...`
- release smoke checks via `make test-release`
- local binary build verification via `go build -ldflags="-s -w -X main.version=local -X main.builtBy=Codex" -o ./k8s-pods-viewer ./cmd/k8s-pods-viewer`

## Notes

- pod usage bars still rely on `metrics.k8s.io` being available in the cluster
- scale actions appear only for pods that belong to a scalable owning workload
- `Exec` currently uses `sh` inside the selected container

## Install

```bash
brew install lavluda/tap/k8s-pods-viewer
```

Or:

