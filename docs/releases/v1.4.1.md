# k8s-pods-viewer v1.4.1

`k8s-pods-viewer` v1.4.1 is a least-privilege compatibility release for users
who can read pods in an EKS or Kubernetes namespace without cluster-wide node
access.

## Highlights

- forbidden node access now enables pod-only mode instead of repeatedly retrying
  the cluster-scoped node informer
- scheduled pods remain visible in pod-only mode even when their nodes cannot be
  listed
- forbidden pod or node watch access now keeps the initial snapshot without
  starting a noisy retry loop
- forbidden pod metrics access now cleanly continues without live usage data
- namespace-scoped least-privilege RBAC requirements and examples are documented

## Release Validation

This release was validated with:

- automated Go tests via `go test ./pkg/... ./cmd/...`
- race detection via `go test -race ./pkg/... ./cmd/...`
- release smoke checks via `make test-release`
- static analysis via `go vet ./pkg/... ./cmd/...`
- local binary build verification via `go build ./cmd/k8s-pods-viewer`

## Notes

- pod `list` and `watch` access is still required in each namespace being viewed
- use `--namespace <name>` when access is limited to one namespace; the default
  all-namespaces scope requires cluster-wide pod access
- node pressure details require optional cluster-scoped node access
- live usage bars require optional pod metrics access through `metrics.k8s.io`

## Install

```bash
brew upgrade lavluda/tap/k8s-pods-viewer
```

Or:

