# k8s-pods-viewer v1.2.0

`k8s-pods-viewer` v1.2.0 reshapes the pod view into a more dashboard-like operator surface while tightening status accuracy on live clusters.

## Highlights

- a redesigned top header now shows context, namespace scope, pod counts, unhealthy totals, sort mode, grouping mode, page, and current time in a more structured layout
- the main pod list keeps the grouped workflow from v1.1.0, but now uses wider usage bars, a clearer command panel, and a dedicated `Highlights` rail for hot pods and cluster watch signals
- pod health rendering now prefers current `Running` and `Ready` state over stale restart history, which fixes false yellow states for recovered pods
- the viewer now runs in the normal terminal screen by default for better compatibility with terminals that clip the top rows in alt-screen mode
- `--alt-screen` remains available as an explicit opt-in for terminals that handle alternate screen rendering well
- the README now includes an updated screenshot of the current dashboard-style interface

## Release validation

This release was validated with:

- automated Go tests via `go test ./pkg/... ./cmd/...`
- release smoke checks via `make test-release`
- local release build verification via `make build`
- live-cluster verification against a 3-node k3s cluster using `kubectl`

## Notes

- live pod and node usage still depends on `metrics.k8s.io` being available in the cluster
- restart counts remain visible as metadata, but no longer force warning state when the pod is currently healthy
- pod drill-down actions such as logs, describe, and exec are still future work

## Install

```bash
go install github.com/lavluda/k8s-pods-viewer/cmd/k8s-pods-viewer@v1.2.0
```
