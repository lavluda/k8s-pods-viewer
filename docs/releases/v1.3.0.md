# k8s-pods-viewer v1.3.0

`k8s-pods-viewer` v1.3.0 turns the pod list into a keyboard-driven operational workspace, with direct pod actions, better startup behavior, and clearer navigation.

## Highlights

- selected pods now open a contextual popup menu with `Exec`, `Logs`, `Describe`, `Kill Pod`, `Scale -1`, and `Scale +1`
- `Logs` and `Describe` now render inside a built-in scrollable viewer, while `Exec` opens an interactive shell and returns cleanly to the TUI
- multi-container pods now prompt for a container before running `Exec` or `Logs`
- dangerous actions now use an inline confirmation step with keyboard-selectable `Confirm` and `Cancel`
- keyboard navigation now follows the exact rendered pod order and stays anchored to page 1 during initial load until the user navigates
- startup now seeds pods and nodes before watches begin, which makes initial rendering much more reliable on live clusters
- pod, node, and metrics watch failures now surface as in-app status instead of crashing the viewer
- the README now documents the current interaction model, controls, and pod action workflow

## Release validation

This release was validated with:

- automated Go tests via `GOCACHE=/tmp/eks-pods-view-go-cache go test ./pkg/... ./cmd/...`
- release smoke checks via `make test-release`
- local binary build verification via `go build -ldflags="-s -w -X main.version=local -X main.builtBy=Codex" -o ./k8s-pods-viewer ./cmd/k8s-pods-viewer`

## Notes

- pod usage bars still rely on `metrics.k8s.io` being available in the cluster
- scale actions appear only for pods that belong to a scalable owning workload
- `Exec` currently uses `sh` inside the selected container
