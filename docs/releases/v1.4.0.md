# k8s-pods-viewer v1.4.0

`k8s-pods-viewer` v1.4.0 is a full visual redesign of the terminal UI, moving from the original plain-text layout to a paneled console aesthetic inspired by modern observability tools.

## Highlights

- **Paneled console layout** — every section (pod list, node pressure, highlights, action popup) is now wrapped in a soft-bordered panel with a visible title, giving the screen clear visual structure without losing terminal density
- **KPI header strip** — four bordered metric tiles (Pods, Unhealthy, Cluster CPU, Cluster Memory) sit directly below the top bar so the most important cluster numbers are visible at a glance before scanning the pod list
- **Segmented usage bars** — CPU and memory bars now use solid background-fill segments that are guaranteed single-cell-wide on all monospace fonts, eliminating the double-width rendering glitch that caused header scroll-off in earlier builds
- **Floating action popup** — the pod action menu (Exec / Logs / Describe / Scale / Kill) is now painted as a true overlay over the pod list using ANSI-aware string splicing; the node pressure and highlights sidebar remain fully intact while the popup is open
- **Accent / density / bar-style flags** — three new CLI flags let you tune the visual without touching code:
  - `--accent #RRGGBB` — accent color for selection bars, panel borders, and the logo dot (default `#39D353`)
  - `--density comfortable|compact` — comfortable adds a dim spacing rule between pod rows; compact packs them tighter
  - `--bar-style segmented|solid|gradient` — bar fill style
- **Node pressure request-marker tick** — a `|` tick on node CPU/MEM bars marks the assigned-requests position relative to allocatable, matching the information shown in the previous text layout
- **Keyboard shortcut chips** — the footer now renders shortcuts as `kbd`-style bordered chips (`↑↓ nav  ←→ page  enter actions …`) instead of a plain help string

## Release validation

This release was validated with:

- automated Go tests via `go test ./cmd/... ./pkg/...`
- release smoke checks via `make test-release`
- local binary build verification via `go build -ldflags="-s -w -X main.version=local -X main.builtBy=Codex" -o ./k8s-pods-viewer ./cmd/k8s-pods-viewer`
- visual verification against the live cluster screenshot at multiple terminal widths (≥150 cols for the side panels, <150 cols single-column fallback)

## Notes

- pod usage bars still rely on `metrics.k8s.io` being available in the cluster
- scale actions appear only for pods that belong to a scalable owning workload (`Deployment`, `StatefulSet`, `ReplicaSet`, `ReplicationController`)
- the floating popup requires a terminal ≥150 columns wide to use the right-column position; on narrower terminals the popup appears inline
- `--alt-screen` mode is unchanged and still recommended for terminals that clip the first rows in inline mode
